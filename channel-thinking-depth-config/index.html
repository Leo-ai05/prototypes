渠道配置模型深度思考深度配置优化

目标是在不重构现有“渠道管理”结构的前提下，为 AI 应答模型增加可控的“深度思考深度”配置，保障运营可按渠道调优回答质量，并通过模型能力标识避免误配。

作用页：`/ChannelManage` 新增字段：`thinkingDepth` 联动页：平台模型管理 / 验证记录

方案总览

新增模型能力标识后，渠道编辑弹窗仅在模型支持时展示“深度思考深度”，避免运营配置无效参数。

配置入口
1 个
位于“AI应答”表单，放在“平台模型”之后、“模型热度”之前。
配置范围
渠道级
支持单渠道新增/编辑，也支持批量修改时选择“保持原值”。
推荐值
标准
低延迟场景建议“浅度”，复杂咨询场景建议“标准/深度”。
C 端影响
0 个新入口
`kefu-web-pc` 不新增前端页面，仅体现在回答耗时和回答细致度变化。

                  1. 当模型不支持深度思考时，字段置灰并强制回写为“关闭”。

                  2. 单渠道新增/编辑时，支持模型默认值为“标准”。

                  3. 批量修改时，首项新增“保持原值”，避免误伤已有渠道策略。

                  4. 选择“深度”后，页面提示回答更细致但耗时可能增加。

                效果说明
                

                  

                    浅度思考
                    
回答更快，适合高频标准问答；内容结构较短。

                  

                  

                    标准思考
                    
默认推荐，平衡回答时延和信息完整度。

                  

                  

                    深度思考
                    
回答更细致，适合复杂咨询，但预计首包耗时更长。

                  

                

                

                  PRD 约束：C 端前端页面不直接展示“思考深度”给用户，仅用于后台配置和复盘，不增加用户认知负担。
